SDK : init

init lets you initialize your Userbase client and automatically log in a user using the last used session.

    userbase.init({ appId, endpoint, keyNotFoundHandler })
      .then((session) => {
        // Userbase client initialized successfully
        
        if (session.user) {
          // there is a valid active session
          console.log(session.user.username)
        }
      })
      .catch((e) => console.error(e))

Parameters

  • settings [object] - Contains your configuration settings.
    • appId [string] - Your App ID you received when you created a Userbase application.
    • endpoint [string | optional] - If you're hosting your own Userbase server, you can provide its endpoint here.
    • keyNotFoundHandler [function | optional] - If a user signs in and the Userbase client can't recover the user's key, by default Userbase displays instructions the user can follow to recover their key. This default behavior can be overridden with this function.
      • username [string] - The username Userbase cannot recover the key for.
      • deviceId [string] - This device's unique ID used when requesting the key from another signed in device.

Result

  • session [object] - Contains information about the last used session.
    • user [object] - If set, this contains information about the logged in user.
      • username [string] - The user's username.
      • key [string] - The user's encryption key.
      • email [string] - The user's email.
      • profile [object] - The user's profile.
    • lastUsedUsername [string] - If set, this is the username of the last user that was logged in.

Errors

  • AppIdMustBeString
  • AppIdCannotBeBlank
  • AppIdNotValid
  • KeyNotFoundHandlerMustBeFunction
  • UserAlreadySignedIn
  • UserCanceledSignIn
  • ServiceUnavailable
