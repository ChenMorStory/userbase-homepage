SDK : signUp

signUp lets you create user accounts for your web app. This API will return a promise that gets resolved once the user account has been created.

    userbase.signUp(username, password, email, profile, showKeyHandler, rememberMe = false, backUpKey = true)
  .then((user) => {
    // user account created
  })
  .catch((e) => console.error(e))

Parameters

  • username [string | Len: 1-50] - The username for the account to be created.
  • password [string | Len: 6-1000] - The password for the account to be created.
  • email [string | optional] - The email for the account to be created.
  • profile [object | optional | Key Len: 1-20 | Val Len: 0-1000 | Keys: 1-100] - The profile for the account to be created. Stored unencrypted. All keys and values must be strings.
  • showKeyHandler [promise | optional] - Upon sign up, by default Userbase displays the user's key instructing the user to save it somewhere safe. This default behavior can be overridden with this function.
    • key [string] - The user's key.
    • rememberMe [boolean] - The rememberMe parameter to signUp.
    • backUpKey [boolean] - The backUpKey parameter to signUp.
  • rememberMe [boolean | optional] - If true, the user's session data and key will be stored in local storage and the user will automatically sign back in when init is called. Defaults to false.
  • backUpKey [boolean | optional] - If true, the user's key will be backed up with their password server-side so the user will only need their password to sign in. Defaults to true.

Result

  • user [object] - Contains information about the new user.
    • username [string] - The user's username.
    • key [string] - The user's key.
    • email [string] - The user's email.
    • profile [object] - The user's profile.

Errors

  • UsernameCannotBeBlank
